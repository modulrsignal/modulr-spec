# Contributing — Modulr Supply Descriptor Contract

---

## What this repository accepts.

Contributions to this repository are welcome in the following areas:

- Corrections to schema definitions or example payloads
- Feedback on descriptor principles
- Suggestions for protocol alignment documentation
- Identified gaps between the descriptor contract and emerging
  standards in AdCP, Prebid, or related ecosystems
- Additional fictional example payloads that illustrate
  descriptor structure

---

## What this repository does not accept.

Contributions in the following areas are out of scope:

- Descriptor generation logic or enrichment algorithms
- Scoring, ranking, or confidence calculation methods
- Publisher ingestion or processing infrastructure
- Execution, bidding, pricing, or transaction systems
- Product roadmap requests
- Commercial feature requests
- Changes that would make the descriptor contract
  dependent on a specific protocol or platform

These areas are intentionally private and are not part of the
public contract.

---

## Schema evolution.

Proposed changes to `schemas/` must include a rationale that
connects the change to one or more principles in
`docs/public-spec/descriptor-principles.md`.

Backward compatibility should be preserved whenever possible.

Schema changes that cannot be traced to a descriptor principle
will not be accepted in v0.1.

---

## How to contribute.

Open an issue before submitting a pull request.
Describe the gap or problem you are addressing.
Reference the relevant doctrine document if applicable.
